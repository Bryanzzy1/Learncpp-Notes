# Compound Data Types

A **compound data type** is a type defined in terms of other existing types (as opposed to fundamental types like `int` or `double`).

A **class type** is any type that is a `struct`, `class`, or `union`. Class types are a specific category of compound types.

This chapter focuses on two foundational compound type families:

- **[[Value Categories]]** — the expression classification system (lvalues and rvalues) that underpins how references and temporaries behave
- **[[Lvalue References]]** — aliases to existing objects
- **[[Pointers]]** — objects that hold memory addresses
